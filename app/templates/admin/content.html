How it works

  • Each content block has a unique key (e.g. hero, services_intro).
  • The body can be plain text or HTML — your choice.
  • The meta JSON field accepts optional structured data (colours, CTA links, image URLs).
  • Blocks are served to the frontend via GET /api/v1/content/{key} — a lightweight, cached endpoint.
  • The frontend falls back gracefully to hardcoded defaults if a block is missing.
  • No admin logic, credentials, or ranking data ever appears in the customer bundle.
